3. DREHANIMATION & MOBILE PERFORMANCE

PROBLEM:

Flip-Card-Animationen können auf mobilen Geräten zu Jank führen (60fps-Drop).


LÖSUNG:


/* GPU-Beschleunigung für smooth Animation */
.flip-card {
  will-change: transform;
  transform: translateZ(0); /* Force GPU rendering */
  backface-visibility: hidden;
}

/* Reduzierte Animation auf Mobile */
@media (max-width: 768px) {
  .flip-card {
    transition: transform 0.4s ease-in-out; /* Schneller auf Mobile */
  }
}

/* Lazy-Loading für Bilder */
<img loading="lazy" src="..." />
PERFORMANCE-OPTIMIERUNGEN:


Bilder komprimieren – WebP-Format für schnelleres Laden

CSS-only Animation – Keine JavaScript-basierte Animation

Intersection Observer – Animationen nur starten, wenn sichtbar

Debouncing – Click-Events mit Verzögerung (verhindert mehrfaches Triggern)


MOBILE-FIRST ANSATZ:


Flip-Cards auf Mobile: Größer (volle Breite minus Padding)

Schriftgröße: Mindestens 16px (Poppins für Body, Montserrat für Headings)

Touch-Ziele: Mindestens 44x44px (Apple HIG Standard)

Keine Hover-States auf Mobile (nur Active-States)
