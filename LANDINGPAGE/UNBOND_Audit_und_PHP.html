🔴 KRITISCH – ohne das kann die Seite nicht live gehen

1. send_to_brevo.php fehlt Muss erstellt werden

Die Landing Page ruft fetch('send_to_brevo.php', ...) auf – diese Datei existiert noch nicht auf dem Server. Ohne sie schlägt jede E-Mail-Anmeldung mit einem Netzwerkfehler fehl.

Lösung: Die fertige PHP-Datei ist weiter unten in diesem Dokument. Einfach als send_to_brevo.php im selben Ordner wie index.html auf Hostinger hochladen.

2. hero-bg.png fehlt Bild hochladen oder URL ersetzen

Die Seite referenziert background: url('hero-bg.png') – lokal, ohne Pfad. Wenn die Datei nicht neben der HTML-Datei auf dem Server liegt, bleibt der Hintergrund einfarbig dunkel (#1a2420) statt das Hintergrundbild zu zeigen.

Lösung: Entweder hero-bg.png auf Hostinger in public_html/ hochladen, oder im CSS durch eine absolute CDN-URL ersetzen.

3. Digistore24-Link ist Platzhalter URL eintragen

Der Link https://www.digistore24.com/PLATZHALTER erscheint 4× in der Seite (Navbar, Hero, E-Book-Section, Result-Modal). Alle klickbaren E-Book-Buttons führen ins Leere.

Lösung: Echten Digistore24-Produkt-Link einsetzen. Suchen & Ersetzen: https://www.digistore24.com/PLATZHALTER → deine echte URL.

4. Score-Diskrepanz: /50 vs. /125 Abstimmen

Der Report (PDF/HTML) zeigt „___ / 125 Punkte". Die Landing Page zeigt im Result-Modal „Dein Score: X / 50". Das verwirrt Nutzerinnen, die beides sehen.

Ursache: 25 Fragen × max. 2 Punkte pro Frage in der JS-Logik = 50. Der Report hingegen rechnet 25 Fragen × max. 5 = 125.

Lösung Option A: In generateProfile() Zeile 3569 den Score mit 2,5 multiplizieren: const reportScore = Math.round(totalScore * 2.5); – dann in der Anzeige reportScore / 125 zeigen.

Lösung Option B: Den Report auf /50 umstellen (einfacher).

🟡 EMPFOHLEN – verbessert Qualität und Conversion

8. Score an Brevo übergeben Empfohlen

Aktuell übergibt handleOptIn() an die PHP: email, zustand, profil – aber nicht den totalScore. Damit kann Brevo keine score-basierte Segmentierung machen.

Fix: In handleOptIn() (Zeile ~3693) die Variable _currentScore ergänzen und im fetch-Body mitsenden. Dazu in generateProfile() Zeile 3567 global speichern:

// In generateProfile(), nach Zeile 3567: window._currentScore = totalScore; // global merken // In handleOptIn(), im JSON.stringify Body ergänzen: body: JSON.stringify({ email: email, zustand: _gabelZustand, profil: _matchedProfileKey || 'unbekannt', score: window._currentScore || 0 // NEU })

9. og:image Meta-Tag fehlt Empfohlen für Social Sharing

Beim Teilen auf WhatsApp, Instagram etc. erscheint kein Vorschaubild. In den <head> einfügen:

<meta property="og:image" content="https://unbond.de/og-image.jpg"> <meta property="og:url" content="https://unbond.de">

Bild: 1200×630px, zeigt Logo + Tagline. Auf Hostinger hochladen.

10. Favicon fehlt Empfohlen

Im Browser-Tab erscheint kein Icon. In den <head> einfügen:

<link rel="icon" type="image/png" href="favicon.png">

11. Kontakt-Link im Footer ist # Ausfüllen

Footer: <a href="#">Kontakt</a> → ersetzen durch mailto:milena@unbond.de oder eine Kontaktseite.

Die Datenschutzerklärung hat bereits einen Hinweis auf Google Fonts. Für maximale DSGVO-Sicherheit können die Fonts lokal auf Hostinger gespeichert werden. Tool: google-webfonts-helper.herokuapp.com – die Fonts herunterladen, hochladen, im CSS referenzieren.

Der fehlende File: send_to_brevo.php

Diesen Code als Datei send_to_brevo.php speichern und in public_html/ (neben index.html) auf Hostinger hochladen.

Vor dem Upload: DEIN_BREVO_API_KEY durch deinen echten API-Key ersetzen (Brevo Dashboard → Einstellungen → API-Keys) und DEINE_LISTEN_ID durch die Zahl deiner Brevo-Kontaktliste.

<?php // ============================================================ // UNBOND – Brevo Opt-In Handler // Datei: send_to_brevo.php // Hochladen nach: public_html/ (gleicher Ordner wie index.html) // ============================================================ // ── Konfiguration ────────────────────────────────────────── define('BREVO_API_KEY', 'DEIN_BREVO_API_KEY'); // aus Brevo Dashboard define('BREVO_LIST_ID', 1); // Zahl deiner Brevo-Liste define('ALLOWED_ORIGIN', 'https://unbond.de'); // deine Domain // ── CORS ────────────────────────────────────────────────── header('Access-Control-Allow-Origin: ' . ALLOWED_ORIGIN); header('Access-Control-Allow-Methods: POST, OPTIONS'); header('Access-Control-Allow-Headers: Content-Type'); header('Content-Type: application/json; charset=UTF-8'); if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') { http_response_code(204); exit; } if ($_SERVER['REQUEST_METHOD'] !== 'POST') { echo json_encode(['success'=>false,'error'=>'Ungültige Anfrage']); exit; } // ── Eingabe lesen & validieren ──────────────────────────── $raw = file_get_contents('php://input'); $data = json_decode($raw, true); $email = isset($data['email']) ? filter_var(trim($data['email']), FILTER_SANITIZE_EMAIL) : ''; $profil = isset($data['profil']) ? htmlspecialchars(trim($data['profil']), ENT_QUOTES) : 'unbekannt'; $zustand= isset($data['zustand']) ? htmlspecialchars(trim($data['zustand']), ENT_QUOTES) : ''; $score = isset($data['score']) ? intval($data['score']) : 0; if (!filter_var($email, FILTER_VALIDATE_EMAIL)) { echo json_encode(['success'=>false, 'error'=>'Ungültige E-Mail-Adresse']); exit; } // ── Brevo API: Kontakt anlegen / aktualisieren ──────────── $payload = [ 'email' => $email, 'updateEnabled' => true, 'listIds' => [BREVO_LIST_ID], 'attributes' => [ 'PROFIL' => $profil, 'ZUSTAND' => $zustand, 'SCORE' => $score, 'TEST_DATUM'=> date('Y-m-d') ] ]; $ch = curl_init('https://api.brevo.com/v3/contacts'); curl_setopt_array($ch, [ CURLOPT_POST => true, CURLOPT_POSTFIELDS => json_encode($payload), CURLOPT_RETURNTRANSFER => true, CURLOPT_HTTPHEADER => [ 'Content-Type: application/json', 'Accept: application/json', 'api-key: ' . BREVO_API_KEY ], CURLOPT_TIMEOUT => 15 ]); $response = curl_exec($ch); $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE); $curlError = curl_error($ch); curl_close($ch); // ── Antwort auswerten ──────────────────────────────────── // 201 = Kontakt neu angelegt, 204 = Kontakt aktualisiert → beides OK if (in_array($httpCode, [201, 204])) { echo json_encode(['success'=>true]); exit; } // Brevo-Fehlerdetails auslesen $brevoData = json_decode($response, true); $errMsg = isset($brevoData['message']) ? $brevoData['message'] : 'Fehler beim Speichern'; // Bekannte Fehlercodes abfangen if ($httpCode === 400 && isset($brevoData['code']) && $brevoData['code'] === 'duplicate_parameter') { // E-Mail schon in der Liste – trotzdem Erfolg anzeigen echo json_encode(['success'=>true]); exit; } http_response_code(500); echo json_encode(['success'=>false, 'error'=>$errMsg]); ?>

Domain mit Hostinger verbinden

Sobald die Dateien hochgeladen sind:

  1. In hPanel: Domains → unbond.de → DNS / Nameserver
  2. Prüfen ob A-Record auf die Hostinger-IP zeigt (hPanel zeigt sie dir)
  3. Wenn Domain woanders registriert: Nameserver auf Hostinger-NS umstellen
  4. SSL: hPanel → SSL → Let's Encrypt aktivieren (kostenlos, automatisch)
  5. Warte 15–60 Min. auf DNS-Propagation
  6. Test: https://unbond.de öffnen → Landing Page erscheint
  7. Test: Opt-In-Formular ausfüllen → Brevo-Dashboard prüfen ob Kontakt angelegt wurde
